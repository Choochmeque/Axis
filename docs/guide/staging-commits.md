# Staging & Commits

## Staging Changes

### Stage Individual Files
- Click the **+** button next to a file to stage it
- Or select multiple files and click **Stage Selected**

### Stage Hunks
- Click on a file to view its diff
- Click **Stage Hunk** to stage individual changes within a file
- This allows you to commit only specific parts of your changes

### Stage All
- Click **Stage All** to stage all modified files

## Unstaging

- Click the **-** button next to a staged file to unstage it
- Or click **Unstage All** to unstage everything

## Creating Commits

1. Stage your changes
2. Enter a commit message in the text field
3. Click **Commit** or press `Cmd/Ctrl + Enter`

### Commit Message Tips
- First line: Brief summary (50 chars or less)
- Leave a blank line
- Additional details if needed

### Amend Last Commit
- Check **Amend** before committing to modify the previous commit
- Useful for fixing typos or adding forgotten files
